# Different Series Chip Library Switching Process

1. Replace the library files in the Hardware folder in Keil.

    For example, when using the F1 series, replace with c files prefixed with stm32f1xx_hal. The ll library can be omitted.
    
    Don't forget to add system_stm32f1xx.c (CMSIS file) and startup_stm32f103xe.s (startup file, select ARM version).
    
    F1 startup file path example: `Hardware\STM32F1XX\CMSIS\Device\ST\STM32F1xx\Source\Templates\arm`, select the startup file corresponding to your chip and capacity.
    
    F4 is similar, all files are under the Hardware folder. You can refer to the file additions in SGA examples.

2. In Options for Target, under the C/C++ menu, modify the preprocessor macro definitions.

    For example, when using STM32F407ZGT6, modify to `USE_HAL_DRIVER,STM32F407xx,STM32F4_SGA_ENABLE`. This switches the SGA library to the F4 version.
    
    For STM32F103XXXX, use `USE_HAL_DRIVER,STM32F103xE,STM32F1_SGA_ENABLE`. This switches the SGA library to the F1 version.

3. In Options for Target, under the Device menu, change to the corresponding chip for debugging and downloading.

You can refer to [PoisonNF/SGA-SampleCode: SGA库的示例代码 (github.com)](https://github.com/PoisonNF/SGA-SampleCode/tree/main), open the corresponding chip project, and add files by comparing with the left-side file list.
